# Vehicle Service Management System

## Overview

This is a database-driven web application designed to manage vehicle servicing operations efficiently.

The system handles customer records, vehicle details, service bookings, payments, and feedback using a structured relational database.

The system ensures data consistency and integrity using relational database design, constraints, and transactions.

---

## Features

* User authentication (Login/Signup)
* Vehicle registration
* Service booking and tracking
* Mechanic assignment
* Payment and invoice system
* Service history tracking
* Feedback system

---

## Tech Stack

* Frontend (HTML, CSS, JavaScript)
* Backend (Node.js / Express)
* MySQL (Database)
* Figma (UI Design)

---

## Database Concepts Used

* Joins
* Views
* Stored Procedures
* Triggers
* Transactions
* Indexing

---

## Project Structure

* frontend/ → User interface
* config/ → Configuration files
* routes/ → API routes
* server.js → Backend entry point
* database.sql → Database schema and queries

---

## How It Works

1. User registers and logs in
2. Adds vehicle details
3. Books a service request
4. System assigns a mechanic and tracks service
5. Payment and invoice are generated
6. Service history and feedback are stored
